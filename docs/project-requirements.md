# 知识表达视频引擎 — 项目需求与路线图

> 本文档是项目顶层需求和开发路线图的唯一权威来源。
> 由五方讨论（Codex / Hermes / Gemini / Claude / User）于 2026-05-30 确定。

## 项目定位

**知识表达视频引擎** — 将知识文案快速转化为动画视频的工具。

不是视频模板系统，不是 PPT 替代品。是一个 JSON Spec 驱动的、SVG 动画渲染引擎，专注于把复杂的 IC/CS/科技概念用动画讲清楚。

## 用户画像

- IC 验证工程师（硕士即将毕业）
- 主要发布平台：小红书 IC 圈
- 这是事业（副业），不是业余项目
- 工作模式：导演型 — 定方向、看辩论、验收成品

## 三类内容

| 类型 | 描述 | 动画深度 | 示例文案 |
|------|------|----------|----------|
| **类型1** | IC 技术深潜 | 重：时序波形、电路行为、协议动画 | "为什么异步复位总要强调同步释放" |
| **类型2** | IC 职业/方法论 | 轻：标题动画、关键句强调、对比布局 | "IC验证工程师真正值钱的能力" |
| **类型3** | AI/科技概念科普 | 中：层级图、概念对比、动态连线 | "大模型、Agent、Skill 有什么区别" |

## 完整生产管线（目标态）

```
文案输入 → AI生成分镜 → 五方讨论 → 用户审核 → 自动渲染(动画+语音+字幕)
    → 五方审核 → 用户验收+调整界面 → 导出视频 → 发布
```

**当前阶段**：动画引擎（上述管线第 5 步的动画部分）。AI 分镜生成、语音、字幕、编辑器、发布是后续工作。

## 已完成（Phase 1-5）

| 阶段 | 内容 | 关键能力 |
|------|------|----------|
| Phase 1 | 数字时序 | `propagateSignal` — 信号传播动画 |
| Phase 2 | 扩展原语 | latch, fanout, glitch, metastability |
| Phase 3 | 流水线 | 5级网格 + Stall/Forward/Flush |
| Phase 4 | 缓存 | 4路组相联 + 地址分解 + Bezier |
| Phase 5 | 分支预测 + 虚拟内存 | BTB + flush动画 / TLB + 页表遍历 |

## 路线图（2026-06 ~ 2026-08）

### Month 1：波形 + 文本

| 原语 | 服务类型 | 说明 |
|------|----------|------|
| **TimingDiagram** | 类型1 | 标准波形视图（clk/rst/data/valid/ready），复用 Phase 1 |
| **CircuitWaveformLinker** | 类型1 | 波形边沿绑定电路节点，时序指针触发电路动画，TimingDiagram 插件 |
| **TextEmphasis** | 类型2 | 关键句浮现、放大、变色、卡点节奏 |

### Month 2：层级 + 对比

| 原语 | 服务类型 | 说明 |
|------|----------|------|
| **LayeredArchitecture** | 类型3 | 层级堆叠图，固定网格布局 |
| **ComparisonLayout** | 类型2+3 | 并列对比布局 |

### Month 3：专业补全 + 收尾

| 原语 | 服务类型 | 说明 |
|------|----------|------|
| **CDC 模板包** | 类型1 | 跨时钟域，模板化固定结构 |
| **Callout** | 全部 | 标注系统（箭头指向 + 说明文字） |
| **Legend** | 全部 | 颜色语义图例 |

## 不做（五方共识）

| 砍掉 | 原因 |
|------|------|
| VerificationFlow | PPT 静态模板够用 |
| 通用 FlowGraph | SVG 自动布局在 1280×720 必翻车 |
| 通用 StateMachine | 自环/双向边在 720p 穿帮 |
| 深层微架构（乱序执行/Tomasulo） | 先补短视频表达层 |
| IsometricStack | 仅封面用，不进核心引擎 |

## 设计约束

- **画布**: 1280×720 SVG
- **架构**: JSON Spec → Schedule（纯函数，计算一次）→ State（per-frame，纯函数）→ SVG Render
- **颜色**: 所有颜色从 `src/theme.ts` 的 `THEME` 对象取
- **动效原则**: 动效必须有教学理由（层次/叙事/反馈/状态转换）
- **SVG 层序**: 背景 → 网格 → 数据单元 → 连线 → 信号叠加层 → 标注
